# Git Workshop Quick Reference

## Key words
- **Repository (repo):** the project folder tracked by Git
- **Stage:** select changes for the next commit
- **Commit:** save a checkpoint in the history
- **Push:** send local commits to GitHub
- **Pull:** bring remote changes from GitHub to your computer
- **Branch:** a separate line of work
- **Merge:** combine branch changes

## Minimum workflow
1. Edit a file
2. Save the file
3. Open **Source Control** in VS Code
4. Stage the file
5. Write a commit message
6. Commit
7. Push

## Commit message examples
Good:
- `Add student information to README`
- `Update hobby file`
- `Add weekend plan`

Less useful:
- `changes`
- `update`
- `work`

## What students should see in VS Code
- **Changed file in Source Control:** the file was edited but not committed yet
- **No changed files after commit:** the commit succeeded
- **Branch name in the bottom bar:** current branch
- **Sync / Push / Pull options:** ways to exchange changes with GitHub

## Five rules
1. One small task -> one commit
2. Read the branch name before editing
3. Push when a task is complete
4. Pull if the remote changed
5. Do not panic if Git asks a question; read the message carefully
